# FormaOps site

Сайт-витрина студии FormaOps — студии прикладных ИИ-решений для бизнеса. Сайт объясняет позиционирование студии, первый продукт SheetNorm, будущую продуктовую линейку, пилотное внедрение и форму заявки.

## Запуск

Команды:

    npm install
    npm run dev
    npm run build
    npm run lint

Локальный адрес разработки: http://localhost:3000.

## Где менять контент

- Контакты, навигация и базовые настройки сайта: src/lib/site.ts.
- Список продуктов и статусы: src/lib/products.ts.
- Конфигурации SheetNorm Team/Enterprise: src/lib/packages.ts.

## Форма заявки

Компонент формы находится в src/components/LeadForm.tsx. API route: src/app/api/lead/route.ts.

На текущем этапе заявка:

- валидируется на сервере;
- не принимает файлы;
- сохраняется в data/leads.json;
- получает createdAt и sourcePage, если источник передан с формы.

Каталог data/ добавлен в .gitignore, поэтому заявки не должны попадать в Git.

Перед production нужно подключить реальную доставку заявок: email, Telegram bot, CRM, Supabase или другой согласованный канал. Также нужно настроить защиту от спама, политику хранения заявок и уведомления ответственных.

## Юридические страницы

Страницы /privacy и /terms являются шаблонами и требуют юридической проверки перед публикацией. В них не указаны выдуманные реквизиты и отдельно отмечено, что через форму не нужно отправлять конфиденциальные файлы.

## Production checklist

- заменить placeholder-контакты в src/lib/site.ts;
- подключить канал доставки заявок;
- проверить тексты privacy/terms с юристом;
- определить домен и metadataBase в src/lib/site.ts;
- настроить аналитику только после проверки требований к персональным данным;
- проверить форму, мобильную верстку, SEO-метаданные и сборку.
